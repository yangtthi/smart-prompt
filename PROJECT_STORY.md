# SmartPrompt 项目故事

## 🌟 起源

你是否曾经为写不好提示词而烦恼？是否觉得每次都要重复复制粘贴相似的 Prompt 很麻烦？

在 AI 时代，Prompt（提示词）是我们与 AI 沟通的桥梁。一个好的 Prompt，可以让你获得更精准、更专业的回答。但随着使用的 Prompt 越来越多，管理它们变成了一件头疼的事情。

## 💡 解决什么问题

- **Prompt 散落各处** - 笔记软件、文档、聊天记录里到处都是
- **版本混乱** - 每次修改都要手动备份，不知道哪个是最新版
- **调试困难** - 变量替换需要手动复制替换，体验很差
- **无法对比** - 不知道哪个版本的 Prompt 效果更好

## 🚀 SmartPrompt 诞生

于是，我们打造了 **SmartPrompt** - 一个轻量级、智能化的 Prompt 管理工具。

### 核心理念

> **让提示词管理变得简单而优雅**

### 核心功能

| 功能 | 描述 |
|------|------|
| 📁 本地存储 | 数据保存在本地文件，安全可控 |
| 📝 版本管理 | 每次修改自动记录版本，保留历史 |
| 🔧 变量替换 | `{{变量}}` 语法，调试预览一键搞定 |
| 🤖 AI 调试 | 集成 DeepSeek API 在线测试效果 |
| ⭐ 评分系统 | 1-5 星评分，记录每个版本的实际效果 |
| 🎨 主题切换 | 明暗主题，随心所欲 |
| 📤 导入导出 | JSON 格式，迁移无忧 |

### 技术栈

- **后端**: Java 8 + SpringBoot 2.7
- **前端**: 原生 HTML/CSS/JavaScript（无框架依赖）
- **存储**: 本地文件系统（JSON）
- **AI**: DeepSeek API

## 📖 使用场景

### 场景一：提示词工程师小李
> 小李需要管理几十个不同场景的 Prompt，有写文章的、代码审查的、数据分析的...
> 使用 SmartPrompt 后，他可以按分类组织 Prompt，每次修改都有历史记录，还能直接在线调试看效果。

### 场景二：产品经理小王
> 小王经常需要优化 AI 生成的文案，通过评分功能，他可以对比不同版本的输出，选出最好的那个。

### 场景三：开发者老张
> 老张把常用的代码注释模板、API 文档生成模板都整理在 SmartPrompt 里，用 `{{函数名}}`、`{{参数}}` 变量，一次编辑，多处使用。
